# 📐 Product Principles
# 📐 产品原则

Version: 0.1

Status: Active

---

# 🎯 Purpose
# 🎯 文档目的

This document defines the product principles of Echendia OS.

本文定义 Echendia OS 的产品设计原则。

Every future feature should follow these principles.

未来所有产品功能都应遵循这些原则。

If a feature violates these principles, we should carefully reconsider whether it should be built.

如果某项功能违反这些原则，我们应认真评估它是否值得开发。

---

# 1️⃣ AI Before Software
# 1️⃣ AI 优先，而不是软件优先

Users should interact with AI instead of learning dozens of software interfaces.

用户应该与 AI 交互，而不是学习几十种不同的软件。

Software should become invisible.

软件应该隐藏在 AI 后面。

---

# 2️⃣ Memory Before Intelligence
# 2️⃣ 记忆优先于智能

A smarter AI is useful.

An AI that remembers is indispensable.

更聪明的 AI 很有用。

真正不可替代的是能够长期记住用户的 AI。

Long-term memory is one of the foundations of Echendia OS.

长期记忆是 Echendia OS 最重要的基础能力之一。

---

# 3️⃣ Execution Before Conversation
# 3️⃣ 执行优先于聊天

AI should do more than answer questions.

AI should help users complete real work.

AI 不应止步于回答问题。

AI 应真正帮助用户完成工作。

Real execution creates real value.

真正的执行，才能真正创造价值。

---

# 4️⃣ Simplicity Over Complexity
# 4️⃣ 简单胜于复杂

Technology should reduce complexity instead of creating more complexity.

技术应该减少复杂性，而不是增加复杂性。

Users should focus on their goals.

AI should handle the complexity behind the scenes.

用户只需要关注目标。

复杂的事情交给 AI。

---

# 5️⃣ Time Is The Product
# 5️⃣ 时间才是真正的产品

We do not sell AI.

We do not sell software.

We help users get their time back.

我们卖的不是 AI。

也不是软件。

我们真正帮助用户的是：

把时间重新还给用户。

Every feature should save users time.

每一个功能，都应该帮助用户节省时间。

---

# 6️⃣ Grow Together
# 6️⃣ 与用户共同成长

AI should become more valuable over time.

AI 应随着时间不断成长。

The more it understands its users, the more useful it becomes.

AI 越了解用户，就越有价值。

---

# ❤️ Product Promise
# ❤️ 产品承诺

Every feature should answer at least one question:

每一个新功能都必须回答下面至少一个问题：

- Does it save users time?
- 它是否帮助用户节省时间？

- Does it reduce complexity?
- 它是否减少复杂性？

- Does it understand users better?
- 它是否更加理解用户？

- Does it help users accomplish real work?
- 它是否真正帮助用户完成工作？

If the answer is no, we should rethink the feature.

如果答案都是否，那么我们应该重新考虑这个功能是否值得开发。

---

# 🚀 Our Product Philosophy
# 🚀 产品哲学

We don't build more software.

We build an AI that understands people and helps them accomplish more meaningful work.

我们不是在打造更多的软件。

我们是在打造一个真正理解用户，并帮助用户完成更有价值工作的 AI。
