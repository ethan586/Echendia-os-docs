# 📚 Documentation Style Guide
# 📚 文档编写规范

Version: 1.0

Status: Active

Last Updated: 2026

---

# 🎯 Purpose
# 🎯 目的

This document defines the official documentation standard for Echendia OS.

本文定义 Echendia OS 的官方文档编写规范。

All documentation inside this repository should follow this guide.

本仓库所有文档均应遵循本规范。

---

# 🌏 Language
# 🌏 语言规范

Every document should be bilingual.

所有文档采用中英双语。

English comes first.

英文在前。

Chinese comes second.

中文在后。

Example：

```
# Vision
# 愿景
```

---

# 📄 File Naming
# 📄 文件命名

Use numbers before filenames.

所有正式文档使用编号。

Example：

001-Vision.md

002-Mission.md

003-Core-Values.md

---

# 🏗 Folder Structure
# 🏗 文件夹结构

Each folder should represent one domain.

每个目录只负责一个领域。

Example：

Database

Architecture

Business

Agent

---

# ✍ Writing Style
# ✍ 编写风格

Write concise.

保持简洁。

Explain why before how.

先解释为什么。

Then explain how.

再解释如何实现。

Avoid unnecessary words.

避免冗余描述。

---

# 🤖 AI Friendly
# 🤖 AI 可读性

Documents should be readable by both humans and AI.

所有文档同时面向人类和 AI。

Keep titles consistent.

标题保持统一。

Use Markdown.

统一采用 Markdown。

---

# 🚀 Principle
# 🚀 原则

Documentation first.

文档优先。

Discussion second.

讨论其次。

Implementation third.

最后才开始开发。
