## What MCP Is
## MCP 是什么

MCP is a protocol that allows AI systems to connect with external tools, data sources, and workflows through a structured interface.

MCP 是一种协议，让 AI 系统能够通过结构化接口连接外部工具、数据源和工作流。

It can give an AI product controlled access to files, repositories, databases, browsers, calendars, communication tools, and business systems.

它可以让 AI 产品以可控方式访问文件、代码仓库、数据库、浏览器、日历、沟通工具和业务系统。

---

## Why MCP Matters To Echendia
## 为什么 MCP 对 Echendia 重要

Echendia OS is not intended to be only a chat interface.

Echendia OS 不应只是一个聊天界面。

It should understand context, coordinate tools, remember important information, and help users execute.

它应理解上下文、协调工具、记住重要信息，并帮助用户执行。

MCP can become one of the connection layers between Echendia OS and the user's real working environment.

MCP 可以成为 Echendia OS 与用户真实工作环境之间的连接层之一。

The value is not in adding many integrations.

价值不在于堆叠大量集成。

The value is in connecting the right tools with clear permissions and observable behavior.

价值在于以清晰权限和可观测行为连接正确的工具。

## Debugging Tools
## 调试工具

MCP debugging should be treated as part of the engineering system.

MCP 调试应被视为工程体系的一部分。

- Mcpsnoop: captures and inspects MCP protocol traffic, similar to Wireshark for MCP.
- Mcpsnoop：抓取并检查 MCP 协议通信，类似 MCP 领域的 Wireshark。

- MCP Inspector: inspects MCP servers, tools, resources, and protocol behavior during development.
- MCP Inspector：在开发过程中检查 MCP 服务、工具、资源和协议行为。

## Adoption Principle
## 采用原则

Do not connect tools before the product reason is clear.

不要在产品理由清晰之前连接工具。

Every MCP integration should answer:

每个 MCP 集成都应回答：

- What user pain does it reduce?
- 它减少了什么用户痛点？

- What data or action does it expose?
- 它暴露了什么数据或操作？

- What permissions does it require?
- 它需要什么权限？

- How will it be observed and debugged?
- 它将如何被观测和调试？
