## Purpose
## 目的

This document defines the MVP hypothesis for Echendia OS v0.1.

本文定义 Echendia OS v0.1 的 MVP 假设。

本文只聚焦第一版产品形态与范围。

---

## Product Shape
## 产品形态

Echendia OS v0.1 is not AI Chat.

Echendia OS v0.1 不是 AI Chat。

It is a Founder Operating System.

它是 Founder Operating System。

Chat is the interaction method, not the product itself.

Chat 是交互方式，而不是产品本身。

The product goal is:

产品目标是：

> Help founders translate long-term goals into daily execution.

> 帮助 Founder 将长期目标转化为每天的执行。

当前 MVP 暂定采用 7 个 Workspace。

## Founder Core
## Founder 核心系统

The seven Workspaces belong to Founder Core.

以上七个 Workspace 属于 Founder Core。

Founder Core should remain universal.

Founder Core 应保持通用。

It should apply to most founders.

它应适用于绝大多数 Founder。

---

## Personal Layer
## 个性化层

Above Founder Core, each user has a Personal Layer.

Founder Core 之上，存在 Personal Layer。

The Personal Layer stores each user's own data and configuration.

Personal Layer 保存每位用户自己的数据与配置。

Examples include:

例如：

- Work shifts
- 工作班次

- Meal times
- 用餐时间

- Fitness goals
- 健身目标

- English learning goals
- 英语学习目标

- Financial accounts
- 财务账户

- Daily habits
- 日常习惯

These belong to user configuration.

这些属于用户配置。

They should not be hardcoded into product logic.

不应写死到产品逻辑。

## Product Principles
## 产品原则

- Chat is the interface, not the product.
- Chat 是入口，不是产品。

- Mission drives execution.
- 长期目标驱动每日执行。

- Founder Core should remain universal.
- Founder Core 保持通用。

- Personal Layer should remain configurable.
- Personal Layer 保持可配置。

- Build for daily use, not occasional use.
- 产品应每天都值得打开。
